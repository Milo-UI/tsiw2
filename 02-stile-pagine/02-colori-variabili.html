Creare una palette di colori e usare le variabili CSS

Una mini guida pratica per capire come passare da una palette di colori a un sistema coerente di token usabile in tutto il progetto.

1) Cos’è una palette (in pratica)

Una buona palette serve a mantenere coerenza visiva, migliorare leggibilità e accessibilità, e rendere il progetto più facile da mantenere. In un sito moderno, i colori diventano un piccolo “sistema” e non una collezione casuale.

Idea chiave: prima scegli i colori base, poi li trasformi in un sistema di utilizzo (token).

2) Colori primitivi: la materia prima

I colori primitivi sono i blocchi base del tuo progetto: brand, accenti, grigi e colori di feedback. Non descrivono ancora dove verranno usati.

3) Token semantici: come userò quei colori

I token semantici descrivono un ruolo: sfondo, testo, bordi, CTA primaria, ecc. Sono quelli che userai davvero nei componenti UI.

Regola d’oro: nei componenti usa token semantici, non i primitivi direttamente.

4) Perché usare le variabili CSS

  • Coerenza: stessi colori ovunque senza errori manuali.
  • Velocità: cambi palette in un punto solo.
  • Scalabilità: più facile mantenere progetti grandi.
  • Temi: base perfetta per light/dark mode.
